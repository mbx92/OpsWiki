OpsWiki — Panduan Pengguna

Technical knowledge base untuk tim DevOps & IT — wiki, SOP, troubleshooting, snippets, dan tools.

1. Pengenalan

OpsWiki adalah platform basis pengetahuan teknis untuk tim DevOps, SRE, dan IT. Semua dokumentasi operasional — runbook, SOP, catatan troubleshooting, perintah shell, dan dokumentasi proyek — dapat dikelola dalam satu workspace.

2. Memulai

2.1 Login

  1. Buka URL aplikasi (mis. https://opswiki.example.com)
  2. Klik Log in
  3. Masukkan email dan password
  4. Setelah login, Anda diarahkan ke Dashboard
Akun harus aktif dan email terverifikasi sebelum bisa mengakses modul utama.

2.2 Registrasi

Jika admin mengaktifkan pendaftaran publik (SAAS_REGISTRATION_ENABLED=true):

  1. Buka halaman Register
  2. Isi: nama, email, password, dan nama workspace
  3. Workspace baru dibuat otomatis dengan paket Free
  4. Anda menjadi Owner workspace tersebut

2.3 Workspace & multi-tenant

Setiap organisasi memiliki workspace terpisah. Data antar workspace tidak saling terlihat.

  • Domain utama — login, marketing, admin platform
  • Subdomain workspace — {slug}.opswiki.example.com
  • Custom domain Pro — domain sendiri yang diarahkan ke workspace

4. Wiki

Wiki adalah inti dokumentasi OpsWiki. Setiap halaman ditulis dalam Markdown dan dirender sebagai HTML.

Membuat halaman wiki

  1. Sidebar → Wiki → Create
  2. Isi title, slug, summary, content (Markdown), category, tags, status, visibility
  3. Klik Save

Status halaman

StatusKapan digunakan
DraftMasih ditulis, belum siap
ReviewMenunggu review rekan
TestedSudah diverifikasi di staging
ProductionDokumentasi resmi/live
DeprecatedTidak lagi berlaku
ArchivedDisimpan untuk arsip

Visibility

LevelSiapa bisa melihat
PrivateAnggota workspace dengan izin wiki
InternalSemua anggota workspace (tanpa link publik)
PublicSiapa saja via link share & portal publik

Riwayat versi

Setiap penyimpanan edit membuat snapshot versi. Akses via History → lihat versi lama → Restore jika perlu.

Relasi konten

Saat membuat/mengedit halaman, tautkan ke: halaman wiki lain, SOP, troubleshooting, project, atau snippet. Relasi muncul di Knowledge Graph.

5. Books (Rak Dokumentasi)

Books mengelompokkan halaman wiki menjadi satu set dokumentasi berurutan. Pro

Membuat book baru

Cara 1 — Book kosong
  1. Sidebar → Books → New Book
  2. Isi Title, Description, Category, Status
  3. Klik Create Book
  4. Tambahkan halaman via import atau pilih halaman wiki standalone
Cara 2 — Import file
  1. Sidebar → Books → Import MD/HTML
  2. Isi nama book, description, category
  3. Upload file .md, .html, .zip, atau pilih folder lokal
  4. Setiap file menjadi satu halaman wiki dalam book

Mengelola book

AksiDeskripsi
EditUbah title, description, category, status
ShareAtur visibilitas publik

Add existing pagesPilih halaman wiki standalone

Remove bookHapus book (halaman wiki tetap ada)

6. Snippets

Perpustakaan perintah, query, atau kode yang sering dipakai.

  1. Sidebar → Snippets → Create
  2. Isi title, platform (bash, docker, kubernetes, sql, dll.), content, description, tags
  3. Klik Copy untuk menyalin ke clipboard
  4. Tandai Favorite untuk akses cepat di Dashboard

10. Inbox

Kotak tangkap cepat untuk ide, log error, perintah, atau draft.

  1. Sidebar → Inbox → Create
  2. Isi title, content, type, source, priority (low → urgent)
  3. Dari detail inbox, klik Convert to: Wiki, Snippet, SOP, atau Troubleshooting

11. Knowledge Graph

Sidebar → Knowledge — peta relasi antar wiki, SOP, troubleshooting, project, dan snippet. Berguna untuk audit kelengkapan dokumentasi dan menemukan konten orphan.

12. Assistant (AI)

Chatbot AI untuk menjawab pertanyaan dari basis pengetahuan workspace. Pro

  1. Admin: Settings → Integrations → AI — aktifkan, isi Base URL, API Key, Model
  2. Sidebar → Assistant → ketik pertanyaan → kirim

14. Assets

Upload gambar, PDF, config, arsip (max 10 MB). Opsional tautkan ke halaman wiki. Jika MinIO dikonfigurasi, file disimpan di object storage.

15. Pencarian

Klik Search di header → ketik kata kunci. Hasil dari wiki, snippets, inbox, SOP, troubleshooting, dan projects. Mode Smart menggunakan pencarian semantik untuk wiki.

16. Berbagi & Portal Publik

Share link halaman

  1. Buka halaman wiki → Share
  2. Set visibility ke Public
  3. Salin link: /share/pages/{slug}

Share book

  1. Buka book → Share
  2. Set visibility Public
  3. Opsional: centang Also make all pages in this book public

Portal publik

Menampilkan semua konten public workspace di /portal atau /w/{slug-workspace}. URL portal terlihat di Settings → Workspace.

21. Tips & Praktik Terbaik

Organisasi konten

  • Gunakan Projects sebagai entry point per aplikasi
  • Quick create dari project untuk dokumentasi ter-link otomatis
  • Books untuk manual/panduan panjang yang berurutan
  • Snippets untuk perintah yang sering dipakai ulang
  • Inbox untuk tangkap cepat, konversi saat sudah matang

Workflow dokumentasi

Inbox (draft cepat) ↓ Convert Wiki / SOP / Troubleshooting ↓ Link Project + Knowledge Graph ↓ Share (optional) Portal Publik / Share Link

Keamanan

  • Default visibility: Private
  • Set Public hanya untuk konten yang memang untuk audience eksternal
  • Review portal publik secara berkala di Settings → Workspace
